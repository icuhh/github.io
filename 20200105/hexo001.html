Hexo + Next 博客搭建教程(1)--基础篇

前言

初次使用Hexo来搭建个人博客，踩了不少坑，本教程将详细介绍基于Hexo + Next主题 的博客的环境安装和本地搭建，希望对大家能有所帮助。
（注：本文是只针对Windows平台的搭建教程，其它可前往我的博客查看）

1、 How to 安装本地开发环境

首先，搭建博客所需开发环境和技术储备有以下：
Git官网下载安装
node.js官网下载安装
coding注册账号–>创建仓库
hexo安装hexo

1.1 安装 Node.js
Hexo是一个基于Node.js的快速、简洁且高效的静态站点生成框架，想要安装Hexo，需要先安装Node.js，具体安装步骤这里不详说。
-安装好后会自动配置好环境变量。- Node.js下载地址

1.2 安装 Git
一样前往官网下载相应版本即可，具体使用也不多说了。- Git下载地址

如果Git和Node.js的环境变量都配置好了，可以通过在cmd中确认安装结果。

1.3 安装 Hexo
如果已经安装好node环境，就可以使用npm来安装Hexo

安装完毕后，可以通过hexo version来确认是否安装成功，到这一步，本地环境已完成。

2、 How to 用 Hexo 快速搭建个人博客

2.1 Hexo初始化
首先先新建一个文件夹，用来存放我们个人博客的各种文件，在cmd命令窗口进入该文件夹，并输入以下指令初始化博客：

初始化成功后可以看到

Start blogging with Hexo!

在刚刚新建的文件夹里会生成一系列相应文件，如下：

  • node_modules：是依赖包
  • public：存放的是生成的页面
  • scaffolds：命令生成文章等的模板
  • source：用命令创建的各种文章
  • themes：主题
  • _config.yml：整个博客的配置
  • db.json：source解析所得到的
  • package.json：项目所需模块项目的配置信息

至此，完成以上步骤就在本地进行静态页面生成，编辑，和预览效果了，当然这样也只能是本地访问，要想让别人访问到，可以把项目部署到github或coding上，具体远程部署步骤请留意后续文章将详细介绍，这里不过多讲解，敬请谅解。

2.3 启动本地 Hexo 服务器

现在你可以通过在浏览器输入 http://localhost:4000/. 即可访问到你个人的博客啦~

4、 可以开始写文章了

Hexo撰写博文也不难，分为 post 和 draft 两种，其中 post 存放在 source/_posts 目录下，draft 存放在 source/_drafts 目录下。 post 和 draft 的区别在于前者会被发布到博客，而后者不会被发布，根目录-》source-》_posts 在此文件夹下写文章。
  • 文章格式为md格式，不熟悉md格式的，可以查看 https://www.runoob.com/markdown/md-tutorial.html
